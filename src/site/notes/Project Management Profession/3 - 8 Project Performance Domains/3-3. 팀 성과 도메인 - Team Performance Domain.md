> [!tip] "PM의 역할은 팀원을 통제하는 게 아니라, 팀이 최고의 성과를 낼 수 있는 환경을 만드는 것."

## 1. 한 줄 요약
- 팀 문화 구축, 팀원 역량 개발, 성과 관리, 리더십 발휘를 통해 성공과 팀을 만드는 도메인
- 2-2 원칙의 마인드셋을 실제 팀 운영으로 연결

## 2. 핵심 활동 (Key Activities)
- 팀 헌장 수립 (규칙/가치관/운영 방식 합의)
- 협업 환경 조성 (심리적 안전감 확보)
- 팀원 역량 개발 및 성장 지원
- 성과 평가 및 피드백
- 갈등 관리 → **2-2 참조**
- 리더십 스타일 적용 → **2-6 참조**

## 3. 주요 개념 & 도구
### 3-1. 팀 헌장 (Team Charter)
- 팀이 함께 만드는 "우리 팀 운영 규칙서". 
- PM이 혼자 만드는 게 아니라 **팀원 모두가 참여해서 합의**하는 게 핵심.

- 포함되는 내용:
	- 팀 가치관 및 원칙
	- 의사결정 방식 (누가, 어떻게 결정하나)
	- 회의 규칙 (빈도, 시간, 방식)
	- 갈등 해결 절차
	- 소통 방식 및 채널

- **시험 포인트:** "팀원 간 갈등이 자주 발생한다, 근본 원인이 뭔가" → **팀 헌장이 없거나 합의가 안 된 것.** 해결책은 팀 헌장 수립.

### 3-2. 심리적 안전감 (Psychological Safety)
- 팀원이 실수나 의견 제시에 대한 두려움 없이 자유롭게 말할 수 있는 환경.
	- 없으면: 팀원이 문제 발견해도 말 안 함 → 나중에 대형 사고
	- 있으면: 빠른 문제 발견 + 혁신적 아이디어 나옴

- **시험 포인트:** "팀원들이 문제를 알고도 보고하지 않는다" → 심리적 안전감 부족. PM이 "실수해도 괜찮다"는 문화 만들어야 함.

### 3-3. RACI 차트
역할과 책임을 명확히 하는 도구. 누가 뭘 하는지 헷갈릴 때 씀.

| |R|A|C|I|
|---|---|---|---|---|
|**의미**|Responsible (실행)|Accountable (최종 책임)|Consulted (자문)|Informed (통보)|
|**누가**|실제로 일하는 사람|결과에 책임지는 사람|의견 줘야 하는 전문가|결과만 알면 되는 사람|
|**수**|여러 명 가능|**반드시 1명**|여러 명 가능|여러 명 가능|

- **핵심:** A(Accountable)는 반드시 한 명. 두 명이면 아무도 책임 안 짐.
- **시험 포인트:** "누가 이 결정의 최종 책임자인지 모른다" → **RACI 차트로 명확화.**

### 3-4. 가상 팀
- 원격/분산 팀 관리 시 추가로 신경 써야 할 것들:
	- **소통:** 비동기(Async) 소통 규칙 명확히. 이메일/슬랙/화상회의 언제 쓸지 합의.
	- **문화 차이:** 시간대, 언어, 업무 문화 차이 인식하고 배려
	- **신뢰 구축:** 직접 못 만나니까 의도적으로 팀빌딩 활동 필요
	- **결과 중심 관리:** 출퇴근 시간이 아닌 결과물로 평가

- **시험 포인트:** "가상 팀에서 팀원들이 소외감을 느낀다" → **정기적인 팀 소통 기회 만들고 참여 유도.** 모니터링 강화는 오답.

### 3-5. 팀 발전 단계 적용
- 터크만 모델 → **2-2 참조**

- 단계별 PM 행동 요약:
	- Forming → 방향/규칙 제시, 팀 헌장 수립
	- Storming → 갈등 중재, Collaborate 모드
	- Norming → 지원, 코칭
	- Performing → 위임, 개입 최소화
	- Adjourning → 성과 인정, 교훈 기록, OPA 업데이트

## 4. 다른 도메인과의 연결
- **3-1 개발 접근방식:** Agile은 Self-organizing 팀이 기본. 팀이 스스로 Sprint 백로그 선택.
- **3-2 이해관계자:** 팀원도 이해관계자. 내부 이해관계자 관리.
- **3-5 계획:** 팀 구성과 역할 배분이 계획 도메인과 연결 (RACI).
- **3-7 측정:** 팀 성과 지표 (속도, 생산성, 참여도) 측정.

## 5. 시험 포인트
- **"팀원이 자기 업무 범위 밖이라고 거부한다"** → **Shared Responsibility 부족.** 팀 헌장 재확인, 공동 책임 문화 강조.
- **"신규 팀원이 합류했다"** → 팀이 **Forming 단계로 일시 후퇴** 가능. 정상적인 현상.
- **"PM이 모든 결정을 혼자 내린다"** → **Empowerment 부족.** 팀원에게 권한 위임해야 함.
- **"팀원 성과가 기대에 못 미친다"** → 먼저 **원인 파악 (역량 부족인지, 동기 부족인지, 장애물인지).** 바로 징계는 오답.
- **"Agile 팀인데 PM이 각 팀원에게 일을 직접 할당한다"** → **오답.** Agile 팀은 Self-organizing. 팀이 스스로 선택.
- **"가상 팀 생산성이 낮다"** → 모니터링 강화 (X) → **소통 채널 정비 + 심리적 안전감 확보 (O).**
