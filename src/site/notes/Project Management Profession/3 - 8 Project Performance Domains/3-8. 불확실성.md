# 1. 정의 (Definition)

**불확실성, 리스크, 모호성을 파악하고 대응**하는 활동. 단순히 리스크 관리만이 아니라, 예측 불가능한 환경에서 프로젝트가 나아갈 수 있도록 하는 전반적인 역량.

# 2. VUCA 환경
- 현대 프로젝트 환경의 4가지 특성:

|영문|한국어|의미|
|---|---|---|
|**V**olatility|변동성|빠르고 예측 불가한 변화|
|**U**ncertainty|불확실성|미래를 알 수 없는 상태|
|**C**omplexity|복잡성|상호작용하는 요소가 너무 많음|
|**A**mbiguity|모호성|상황/요구사항 자체가 불명확|

# 3. 불확실성 유형

| 유형                   | 설명                                          |
| -------------------- | ------------------------------------------- |
| **Risk (알려진 모름)**    | 식별은 됐지만 발생 여부 불확실                           |
| **Unknown Unknowns** | 식별조차 못한 리스크. 관리 예비비(Management Reserve)로 대비 |
| **Complexity**       | 요소 간 상호작용이 너무 복잡해 결과 예측 불가                  |
| **Ambiguity**        | 무엇이 불명확한지조차 모름                              |

# 4. 리스크 관련 핵심 개념

**(2-10 원칙과 연계)**
- **리스크 성향 3가지 용어 구분** — 시험 자주 출제!

|용어|의미|
|---|---|
|**Risk Appetite** (리스크 선호도)|조직이 감수하려는 리스크의 성격/양 (전략적 성향)|
|**Risk Tolerance** (리스크 허용범위)|감수 가능한 리스크의 허용 한계치|
|**Risk Threshold** (리스크 임계값)|허용 한계를 넘어서는 특정 수치 기준점|

# 5. 리스크 분석 기법

> 리스크 관리 프로세스 순서:
> **<mark style="background:#fff88f">식별 → 정성적 분석 → 정량적 분석 → 대응 계획</mark>**
> Qualitative에서 우선순위 높게 분류된 리스크만 → Quantitative로 넘어감

## 5-1. **정성적 분석 (Qualitative)**

- 확률-영향 매트릭스 (Probability × Impact Matrix)
- 빠르게 우선순위 분류 → **모든 프로젝트**에 적용
- 역할: **스크리닝** — "어떤 리스크가 중요한가?" 필터링

## 5-2. **정량적 분석 (Quantitative)**

- **Qualitative에서 우선순위가 높게 분류된 리스크**에만 수행 — 시험 출제 포인트!
- 고위험/대형 프로젝트에만 선별 적용
- 역할: **심층 분석** — "얼마나 영향이 크고 확률이 높은가?" 수치화

|기법|설명|
|---|---|
|**EMV** (Expected Monetary Value)|확률 × 금전적 영향. 의사결정 트리에 활용|
|**Decision Tree**|여러 선택지의 EMV 비교해 최적 결정|
|**Monte Carlo Simulation**|수천 번 시뮬레이션으로 결과 분포 예측|

> 💡 EMV 예시: 30% 확률로 -$100K 손실 → EMV = 0.3 × (-100,000) = **-$30,000**

# 6. 리스크 대응 전략 (Risk Response Strategies)

## 6-1. 위협(Threat)에 대한 대응 전략

| 전략 | 설명 |
|---|---|
| **Avoid (회피)** | 리스크 원인을 제거해서 아예 발생 못 하게 함. 프로젝트 계획 변경 |
| **Transfer (전가)** | 리스크를 제3자에게 넘김 (예: 보험, 외주 계약) |
| **Mitigate (완화)** | 리스크 발생 확률 또는 영향을 줄임 |
| **Accept (수용)** | 리스크를 그냥 받아들임. 아래 두 가지로 나뉨 |

## 6-2. Accept의 두 가지 유형 — 시험 자주 출제!

| 유형 | 행동 | 핵심 |
|---|---|---|
| **Active Acceptance** | Contingency Plan(대응 계획)을 미리 세워둠 | 리스크 발생 시 즉시 실행할 계획 존재 |
| **Passive Acceptance** | 아무 행동 안 함. **단, 전략은 문서화** | 리스크 발생하면 그때 팀이 알아서 대응 |

> ⚠️ **Passive Acceptance ≠ 완전히 아무것도 안 함**
> "Doing nothing"은 **프로젝트 계획을 바꾸는 행동을 안 한다**는 뜻.
> 선택한 전략을 **문서화하는 것은 필수**. 리스크 관리에서 모든 결정은 기록이 원칙.
> → C("no action") 아닌 D("문서화 후 발생 시 팀이 대응")가 정답인 이유

## 6-3. 기회(Opportunity)에 대한 대응 전략

| 전략 | 설명 |
|---|---|
| **Exploit (활용)** | 기회가 반드시 실현되도록 적극 조치 |
| **Share (공유)** | 기회를 잘 살릴 수 있는 제3자와 협력 |
| **Enhance (향상)** | 기회 발생 확률 또는 영향을 높임 |
| **Accept (수용)** | 기회가 오면 활용하되 별도 조치는 안 함 |
