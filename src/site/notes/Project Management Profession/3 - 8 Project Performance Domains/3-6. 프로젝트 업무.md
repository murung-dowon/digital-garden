# 1. 정의 

 - 프로젝트 팀이 **프로젝트 프로세스를 수립하고, 물적 자원을 관리하고, 학습 환경을 조성**하는 활동과 기능. 
 - 쉽게 말해 "프로젝트가 실제로 굴러가도록" 하는 일상적 운영 영역.

# 2. 핵심 활동 영역
|활동|내용|
|---|---|
|프로세스 수립|팀이 어떻게 일할지 규칙/절차 정의|
|물적 자원 관리|자재, 장비, 공급물 조달/관리|
|조달 관리|외부 벤더/계약 관리|
|지식 관리|명시적/암묵적 지식 보존 및 공유|
|의사소통 관리|이해관계자와 적절한 정보 흐름 유지|
|변경 모니터링|새로운 작업/변경사항 추적|

# 3. 조달 관리 (Procurement)

## 3-1. 계약 유형 3가지
| 유형                         | 특징                     | 리스크 부담          |
| -------------------------- | ---------------------- | --------------- |
| **Fixed Price (FP)**       | 총액 고정. 범위가 명확할 때 사용    | ==판매자(Seller)== |
| **Cost Reimbursable (CR)** | 실제 비용 + 수수료. 범위 불명확할 때 | ==구매자(Buyer)==  |
| **Time & Materials (T&M)** | 시간+재료 단가 기준. 단기/소규모 작업 | 중간 (둘 다)        |

### 3-1-1. Fixed Price 세부 유형 
- **FFP** (Firm Fixed Price): 가장 일반적. 완전 고정
- **FP-EPA** (Fixed Price + Economic Price Adjustment): 장기계약 시 물가 반영
- **FPIF** (Fixed Price Incentive Fee): 목표 달성 시 인센티브

### 3-1-2. Cost Reimbursable 세부 유형
- **CPFF** (Cost Plus Fixed Fee): 고정 수수료
- **CPIF** (Cost Plus Incentive Fee): 성과 인센티브
- **CPAF** (Cost Plus Award Fee): 평가 기반 보상

# 4. 의사소통 관리
## 4-1. 커뮤니케이션 채널 공식

- 채널수=n(n−1)2채널 수 = \frac{n(n-1)}{2}채널수=2n(n−1)​
- 예) 팀원 5명 → 5×4/2 = **10개 채널**

## 4-2. 커뮤니케이션 방식 3가지

|방식|설명|예시|
|---|---|---|
|**Interactive**|양방향 실시간|회의, 전화|
|**Push**|발신자 → 수신자 (일방향)|이메일, 보고서|
|**Pull**|수신자가 필요시 접근|인트라넷, 공유 문서|

## 4-3. Work Performance 흐름 (WPD → WPI → WPR)

프로젝트 실행 중 데이터가 아래 단계로 정제되어 이해관계자에게 전달됨:

```
[실행] ──▶ WPD ──▶ [분석] ──▶ WPI ──▶ [보고서화] ──▶ WPR ──▶ [이해관계자]
```

| 단계 | 이름 | 설명 | 예시 |
|------|------|------|------|
| **WPD** | Work Performance Data (작업 성과 데이터) | 날것의 원시 데이터. 아직 해석 없음 | "태스크 37개 중 20개 완료", "실제 비용 $50,000" |
| **WPI** | Work Performance Information (작업 성과 정보) | WPD를 분석해 계획 대비 의미 부여 | "일정 10% 지연 중", "CPI = 0.85" |
| **WPR** | Work Performance Report (작업 성과 보고서) | WPI를 이해관계자용 포맷으로 정리한 문서 | Status Report, Dashboard, Update Memo |

### WPR에 해당하는 문서 예시 ✅
- Status Report (상태 보고서)
- Project Update Memo
- Project Recommendations
- Dashboard

### WPR이 **아닌** 것 ❌
- **Project Charter** → 착수(Initiating) 단계 문서. 프로젝트 진행 중 만드는 보고서가 아님

> 📝 **시험 포인트 (Set A - Q16):** "다음 중 WPR의 예시가 아닌 것은?" → **Project Charter**. 나머지(Status Report, Update Memo, Recommendations)는 전부 WPR.


# 5. 지식 관리
| 구분                    | 설명                    |
| --------------------- | --------------------- |
| **명시적 지식 (Explicit)** | 문서화 가능한 지식 (매뉴얼, 보고서) |
| **암묵적 지식 (Tacit)**    | 경험에서 나오는 노하우, 문서화 어려움 |
- **교훈 (Lessons Learned)**: 진행 중 + 종료 시 모두 기록. 조직 프로세스 자산(OPA)에 저장
- 지식 이전은 팀원 교체/이탈 전에 반드시 수행


# 6. 성과 확인 (Outcomes — "잘 되고 있다"의 기준)

PMBOK 7판이 체크하는 이 도메인의 기대 성과:

- ✅ 프로젝트 프로세스가 효율적으로 운영됨
- ✅ 이해관계자와의 의사소통이 원활함
- ✅ 물적 자원이 적시에 적절히 관리됨
- ✅ 조달 계약이 효과적으로 관리됨
- ✅ 팀 학습 및 지식이 조직에 보존됨

- 명시적 vs 암묵적 지식 구분
