### Set A - Q13 
- Project Charter vs Scope document
	- **Charter:** "우리는 'A'라는 가치를 만들기 위해 모였다!" (헌법)
	- **Scope:** "그래서 구체적으로 'A-1, A-2'를 만들 거다." (법률)

### Set A - Q15
- <mark style="background:#fff88f">CPM에 관한 개념</mark> + 각각 하위 ES ~ LF 내용들도 같이 정리해야함. 
- Critical Path method - 주공정법
	- **Early Start (ES):** "빨리빨리!" - 앞에 일 끝나자마자 **가장 빨리 시작**할 수 있는 날짜.
	- **Early Finish (EF):** 제일 빨리 시작해서 끝냈을 때의 날짜.
	- **Late Start (LS):** "최대한 버텨!" - 전체 프로젝트 안 늦추려면 **아무리 늦어도 이때는 시작**해야 하는 날짜.
	- **Late Finish (LF):** 아무리 늦어도 이때는 끝내야 하는 날짜.

### Set A - Q22
- Scope management vs Change control
	- (정리된 노트 기준은 편의상 분류한 것.) ==Scope management는 계획 도메인, Change control은 업무 도메인==.

### Set A - Q29
**Acceptance is a strategy adopted because it is not possible to eliminate all risks from a project. This strategy indicates that the project management team has decided not to change the project management plan to deal with a risk. What action does <mark style="background:#fff88f">passive acceptance</mark> require?**

**D**. Passive acceptance requires no action except to document the strategy, leaving the project team to deal with the risks as they occur.
