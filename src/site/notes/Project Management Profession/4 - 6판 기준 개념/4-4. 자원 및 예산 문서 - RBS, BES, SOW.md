> [!tip] "스폰서가 'resource estimates 보여줘' 하면 → RBS. 'budget 보여줘' 하면 → BES. 범위 기술이 필요하면 → SOW."

# 1. 한 줄 요약

- **RBS** = 자원을 계층적으로 분류한 구조. "누가/뭐가 투입되는지" 자원 관점.
- **BES** = 예산 산정 시트. "얼마나 드는지" 비용 관점.
- **SOW** = 작업 범위 기술서. "무엇을 할 건지" 범위 관점. → 자세한 내용은 **4-1. 조달 관리** 참조.

# 2. RBS (Resource Breakdown Structure) — 자원 분류 체계

PMBOK 6판 Resource Management Knowledge Area의 핵심 도구.

## 개념

자원(Resource)을 **카테고리별로 계층 구조**로 분류한 문서.  
WBS가 "작업"을 쪼개는 것처럼, RBS는 "자원"을 쪼개는 것.

```
프로젝트 전체 자원
    ├── 인적 자원 (Human Resources)
    │       ├── PM
    │       ├── 개발자
    │       └── QA 엔지니어
    ├── 장비 (Equipment)
    │       ├── 서버
    │       └── 테스트 장비
    └── 자재 (Materials)
            ├── 소프트웨어 라이선스
            └── 소모품
```

## 언제 만드나

**Plan Resource Management** 프로세스의 output 중 하나.  
→ 프로젝트 초반 계획 단계에서 자원 범주를 정의할 때 작성.

## 시험 포인트

> "스폰서가 early resource estimates를 보고 싶다" → **RBS draft 공유** ✅  
> "BES(예산 시트)를 공유한다" → ❌ BES는 비용 중심, resource 중심이 아님  
> "SOW를 다시 공유한다" → ❌ SOW는 범위 기술서, 자원 정보 없음

# 3. BES (Budget Estimate Sheet) — 예산 산정 시트

## 개념

> ⚠️ **PMI 공식 표준 용어는 아님.** 시험에서 RBS와의 구분 포인트로 등장.

예산(Budget) 또는 비용(Cost) 관점에서 항목별 예상 금액을 정리한 문서.  
"이 프로젝트에 얼마 들겠나"를 수치로 정리한 것.

## RBS vs BES — 핵심 차이

| | **RBS** | **BES** |
|---|---|---|
| **초점** | 자원 (사람, 장비, 자재) | 비용 (금액, 예산) |
| **질문** | "무슨 자원이 얼마나 필요해?" | "비용이 얼마야?" |
| **PMI 표준** | ✅ 공식 용어 | ❌ 비공식 (일반 용어) |
| **관련 프로세스** | Plan Resource Management | Estimate Costs / Determine Budget |

> 📝 **시험 포인트:**  
> "resource estimates" 요청 → **RBS**  
> "budget/cost 정보" 요청 → **BES 또는 Cost Estimates**

# 4. SOW (Statement of Work) — 작업 범위 기술서

> 자세한 내용 → **[[Project Management Profession/4 - 6판 기준 개념/4-1. 조달 관리 - Project Procurement Management\|4-1. 조달 관리 - Project Procurement Management]]** 참조

## 간단 요약

조달 계약 전 "이 프로젝트/계약에서 무엇을 할 건지"를 기술한 문서.  
범위(Scope) 관점의 문서이므로 자원 정보나 예산 정보는 없음.

| 문서 | 답하는 질문 |
|------|------------|
| **SOW** | 무엇을(What) 할 건가? |
| **RBS** | 어떤 자원이(Who/What resource) 필요한가? |
| **BES** | 얼마나(How much) 드는가? |

# 5. 관련 문제

  - C (SOW 재공유) 오답 이유: SOW는 범위 기술서, 자원 정보 없음
