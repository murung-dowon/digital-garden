>[!tip] PM인 나에게 영향을 주는 요소

### 1. 기업 환경 요인 (Enterprise Environmental Factor) 
- **한 줄 요약:** "내가 통제할 수 없는 제약 조건" (내/외부 모두 포함, 날씨 같은 놈)
- **특징:** PM이나 팀이 통제할 수 없음. 그냥 받아들이고 적응해야 함.
- **예시:**
    - **외부:** 정부 규제(법), 시장 상황, 환율, 업계 표준.
    - **내부:** 조직 문화(빡센지 유연한지), 기존 인프라, IT 시스템 성능.

### 2. 조직 프로세스 자산 (Organizational Process Assets)
- **한 줄 요약:** "조직이 나한테 주는 족보와 도구" (전쟁 나갈 때 주는 보급품)
- **특징:** 조직 내부의 지식, 템플릿, 데이터베이스임. 적극적으로 활용해야 함.
- **예시:**
    - **프로세스/절차:** 문서 템플릿, 승인 절차 가이드라인, 변경 통제 방법.        
    - **지식 베이스:** 예전 프로젝트에서 쓴 **교훈(Lessons Learned)**, 과거 예산 데이터.

### 2-1. 조직 구조 유형 (Organizational Structure) — 내부 EEF

조직 구조는 PM의 권한과 팀원 보고 체계를 결정하는 내부 EEF임.

```
기능 조직 ◀────────── 매트릭스 조직 ──────────▶ 프로젝트화 조직
(Functional)              (Matrix)               (Projectized)

PM 권한 최소              권한 공유               PM 권한 최대
팀원 = 부서 소속          팀원 = 부서 + 프로젝트   팀원 = 프로젝트 소속
부서 매니저가 인사권       둘이 협의               PM이 인사권
```

| | Functional | Weak Matrix | Balanced Matrix | Strong Matrix | Projectized |
|---|---|---|---|---|---|
| **PM 권한** | 거의 없음 | 낮음 | 중간 | 높음 | 최대 |
| **실권자** | 부서 매니저 | 부서 매니저 | 공유 | PM | PM |
| **팀원 보고** | 부서 매니저에게만 | 주로 부서 매니저 | 둘 다 | 주로 PM | PM에게만 |

#### ⚠️ Matrix 조직에서 PM이 해야 할 것 (시험 포인트)
- 팀원이 다른 부서 매니저에게 보고하는 구조 → PM이 **직접 지시 권한 없음**
- 팀원 참여 필요 시 → **사전에 부서 매니저에게 협조 요청** + 일정/기간 미리 공유
- Project Charter 들이밀며 강요 ❌ / 매니저 무시하고 팀원 직접 호출 ❌

> 📝 **시험 포인트 (Set A - Q23):** "팀원들이 다른 매니저에게 보고한다, 미팅 참여시키려면?" → **사전에 해당 매니저에게 협조 요청 + 일정 공유 (C)**

### 3. EEF vs OPA 차이
| **구분**      | **EEF (기업 환경 요인)** | **OPA (조직 프로세스 자산)** |
| ----------- | ------------------ | -------------------- |
| **성격**      | 나를 **구속**하는 것 (제약) | 나를 **도와주는** 것 (자산)   |
| **핵심 키워드**  | 법, 문화, 시장, 시스템     | 템플릿, 과거 기록, 절차, 교훈   |
| **PM의 태도**  | "아... 적응해야겠네 (순응)" | "오예! 이거 써먹어야지 (활용)"  |
| PM의 업데이트 여부 | 불가 (Input으로만 사용)   | 가능 (프로젝트 종료시)        |
